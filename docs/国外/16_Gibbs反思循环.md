# Gibbs反思循环（Gibbs' Reflective Cycle）

## 概述

Gibbs反思循环由学者 **Graham Gibbs** 于1988年提出，是一种六阶段深度反思模型。与Kolb循环相比，Gibbs循环特别增加了**情感维度**（Feelings），使反思更加全面和人性化。广泛用于护理、教育、社会工作等领域的专业反思。

> 📖 来源：Gibbs, G. (1988). *Learning by Doing: A Guide to Teaching and Learning Methods.*

---

## 六个阶段

| 阶段 | 英文 | 中文 | 核心问题 |
|------|------|------|----------|
| ① | Description | 描述 | 发生了什么？ |
| ② | Feelings | 感受 | 我当时有什么感受？ |
| ③ | Evaluation | 评估 | 什么好？什么不好？ |
| ④ | Analysis | 分析 | 为什么会这样？ |
| ⑤ | Conclusion | 结论 | 还能做什么？ |
| ⑥ | Action Plan | 行动计划 | 下次遇到会怎么做？ |

---

## 详细说明

### ① Description（描述）

**目的：** 客观还原事件

**关键问题：**
- 发生了什么？
- 在哪里？什么时候？
- 有谁在场？
- 你做了什么？别人做了什么？

**要点：**
- 只描述事实，不加评判
- 不分析原因，不表达感受（这些留给后面的阶段）
- 像记者一样客观记录

**示例：**
> "周三下午2点，我在部门周会上汇报Q1市场数据。会议室有8个人，包括我的直属上级王总。我在汇报到第3页PPT时，被王总打断，他说'这些数据我上周已经看过了，说点新的'。我愣了大约5秒，然后说'我来补充一下上周没有的新数据'，匆忙翻到后面的页面继续汇报。整个汇报原本计划20分钟，实际用了35分钟。"

---

### ② Feelings（感受）

**目的：** 识别并表达事件中的情绪

**关键问题：**
- 你在事件之前/之中/之后有什么感受？
- 你觉得其他人当时有什么感受？
- 你现在回顾这件事有什么感受？

**要点：**
- 不要跳过这一步！情绪是反思的重要线索
- 具体说出感受的名称（焦虑、尴尬、愤怒、沮丧……）
- 区分"当时"和"现在"的感受——它们可能不同

**示例：**
> "汇报前：紧张但不焦虑，觉得自己准备充分。被王总打断时：瞬间很尴尬，脑子一片空白，感觉脸在发烫。汇报中：焦虑、慌乱，因为被迫跳过了准备好的过渡逻辑。汇报后：沮丧和挫败，觉得自己的努力被否定了。现在回顾：仍然有些不舒服，但也能理解王总的立场——他确实需要新信息。"

---

### ③ Evaluation（评估）

**目的：** 判断这次经历的"好"与"不好"

**关键问题：**
- 什么做得好？（积极的方面）
- 什么做得不好？（消极的方面）
- 有什么亮点或遗憾？

**要点：**
- 好的和不好的都要写，不能只写一面
- 尽量具体，不说"还行"这种模糊评价

**示例：**
```
好的方面：
  ✅ 数据准备充分，质量没问题
  ✅ 被打断后没有失态，迅速调整
  ✅ PPT整体设计专业

不好的方面：
  ❌ 没有提前了解听众的已知信息
  ❌ 汇报结构没有以"新信息"为亮点
  ❌ 被打断后节奏乱了，后面越说越快
  ❌ 超时15分钟
```

---

### ④ Analysis（分析）

**目的：** 深入理解"为什么"

**关键问题：**
- 为什么好的方面做得好？
- 为什么不好的方面出了问题？
- 整体来看，这件事说明了什么？

**要点：**
- 这是最有价值的阶段
- 从多个角度分析（自身、他人、环境、系统）
- 寻找根本原因

**示例：**
> "好的方面之所以好，是因为我在数据准备上花了3小时，做了交叉验证。
>
> 不好的方面的根本原因：① 我假设听众什么都不知道，没有做'听众分析'——王总上周已经看过类似数据，我却重复汇报；② 我的汇报逻辑是'背景→旧数据→新发现'，但王总期待直接说新发现；③ 我没有准备'被提问/打断'的应对预案，导致慌乱。
>
> 更深层的问题：我在向上汇报时，习惯于'自上而下讲完整故事'，但高管更偏好'先说结论/新信息，再展开细节'。"

---

### ⑤ Conclusion（结论）

**目的：** 提炼出通用性的认识

**关键问题：**
- 我从这次经历中学到了什么？
- 还能有什么其他选择？
- 下次类似情况我会怎么做？

**要点：**
- 不只说"下次注意"，要给出具体的改进方向
- 思考是否有更系统性的改变需要做

**示例：**
> "1. 汇报前必须了解听众已知信息，避免重复。
> 2. 向高管汇报应该'先结论后细节'（金字塔原理）。
> 3. 每次汇报准备一个'3分钟版本'作为应急预案。
> 4. 被打断不是否定，而是沟通信号——我应该学会欢迎打断。"

---

### ⑥ Action Plan（行动计划）

**目的：** 制定具体可执行的改进方案

**关键问题：**
- 下次遇到类似情况，我会具体怎么做？
- 需要什么资源/支持？
- 如何衡量改进效果？

**要点：**
- 必须是具体、可执行的
- 有时间节点
- 可以验证效果

**示例：**
```
行动计划：
□ 下周一汇报前，先发消息问王总："您对Q1数据有哪些已了解的信息？"
□ 今晚花30分钟学习《金字塔原理》的汇报结构
□ 每次汇报准备两个版本：完整版（20分钟）和快速版（5分钟）
□ 下次汇报后，请同事反馈我的汇报效果

验证方式：下次汇报不被打断且不超时 = 改进成功
```

## Gibbs vs Kolb 对比

| 维度 | Gibbs反思循环 | Kolb学习循环 |
|------|-------------|-------------|
| **阶段数** | 6个 | 4个 |
| **情感维度** | ✅ 有专门的Feelings阶段 | ❌ 无专门阶段 |
| **适用对象** | 个人深度反思 | 团队学习设计 |
| **结构化程度** | 高，每步有明确问题 | 中，更灵活 |
| **行动导向** | ✅ 有Action Plan阶段 | ✅ 有Active Experimentation |
| **典型应用** | 护理、教育、社工 | 培训设计、组织学习 |

## 应用场景

| 场景 | 说明 |
|------|------|
| **专业反思** | 医疗、教育、社工等行业的定期反思 |
| **个人成长日志** | 每周选一件重要事件用Gibbs循环反思 |
| **团队复盘** | 团队遇到问题后，每个人独立完成Gibbs循环再分享 |
| **导师辅导** | 被辅导者完成Gibbs反思，导师针对性指导 |
| **面试准备** | 用Gibbs结构梳理过往经历的反思 |

---

## Gibbs循环的核心价值

> **Gibbs循环最独特之处在于"Feelings（感受）"阶段。大多数反思模型忽略了情绪，但情绪往往是行为背后的真正驱动力。承认并理解自己的情绪，才能做出真正有效的改变。**

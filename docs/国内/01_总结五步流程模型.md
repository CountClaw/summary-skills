# 总结五步流程模型

## 概述

总结五步流程模型是一种从原始数据到可复用经验的系统性总结方法。它将总结过程拆解为五个递进阶段，确保从海量信息中提炼出真正有价值的知识与行动指引。

---

## 五步流程

### 第一步：收集数据

| 要点 | 说明 |
|------|------|
| 操作 | 回顾主题，收集所有相关数据 |
| 关键 | 数据越充分，越能反映真实情况；没有数据就没有总结 |
| 注意 | 先记录，不评判，避免遗漏 |

### 第二步：提取信息

| 要点 | 说明 |
|------|------|
| 操作 | 对数据进行归类、整理、筛选 |
| 关键 | 根据目标筛选；用多维度确保完整性；反复回顾防止遗漏 |
| 注意 | 提取的信息应与总结目标高度相关 |

### 第三步：梳理与归纳

| 要点 | 说明 |
|------|------|
| 操作 | 将信息纳入不同维度，精加工处理 |
| 关键 | 提取核心概念和关键要点；找出普遍性规律和底层逻辑 |
| 注意 | 避免面面俱到，聚焦核心 |

### 第四步：形成经验

| 要点 | 说明 |
|------|------|
| 操作 | 将总结变成可复用的经验 |
| 核心问题 | 以后再遇到类似事情怎么处理？对我有什么影响？ |
| 注意 | 经验 ≠ 经历，经验是从经历中提取的智慧 |

### 第五步：写下来

| 要点 | 说明 |
|------|------|
| 操作 | 记录归档，长期积累 |
| 关键 | 积累到一定程度后可形成知识体系，产生质的飞跃 |
| 注意 | 定期回看，让总结真正被使用 |

---

## 流程图

```
数据 → 信息 → 普遍性规律/知识 → 道理 → 具体应用
收集   提取      梳理与归纳       形成经验   写下来
```

---

## 核心原则

1. **总结要有目标**：不能面面俱到，什么都抓等于什么都没抓住
2. **从多维度总结**：单一维度的总结是片面的，需要从不同方面切入
3. **总结中一定要有思考**："流水账" ≠ 总结，流水账只是记录发生了什么
4. **总结要具备指导作用**：经历 ≠ 经验，好的总结要能指导未来的行动和决策

---

## 应用场景

- 年终工作总结
- 项目复盘
- 学习总结
- 个人成长回顾
- 旅行/活动总结

---

## 反面示例 vs 正面示例

| 场景 | ❌ 反面 | ✅ 正面 |
|------|---------|---------|
| 旅行总结 | "预算超了，下次注意" | "住宿费占比应控制在 x% 以下，饮食 x% 以下，纪念品 x% 以下" |
| 项目总结 | "项目延期了，团队不够给力" | "需求变更超过3次时，应触发重新评估机制，预留2周缓冲期" |
| 学习总结 | "这门课很有收获" | "掌握了3个核心模型：A用于分析市场，B用于评估风险，C用于制定策略" |

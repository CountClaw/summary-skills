# PDCA循环

## 概述

PDCA循环（又称戴明环 / Deming Cycle）是由质量管理专家 **W. Edwards Deming** 推广的一种持续改进方法论。它通过"计划→执行→检查→改进"四个阶段的循环往复，推动工作和质量的不断提升。

---

## 四个阶段

| 字母 | 英文 | 中文 | 核心问题 |
|------|------|------|----------|
| **P** | Plan | 计划 | 要做什么？怎么做？ |
| **D** | Do | 执行 | 按计划去做 |
| **C** | Check | 检查 | 做得怎么样？结果如何？ |
| **A** | Act | 改进 | 如何优化？下一步怎么办？ |

---

## 详细说明

### P - Plan（计划）

**核心任务：** 制定目标和行动方案

**关键步骤：**
1. 明确目标（SMART原则）
2. 分析现状与目标的差距
3. 制定具体的行动方案
4. 预设衡量标准

**要点：**
- 计划要具体可执行，不能太笼统
- 要考虑可能的障碍和应对方案
- 让相关方达成共识

**示例：**
> "目标：本月将客户投诉率从5%降至2%。方案：① 建立投诉分类台账；② 对高频投诉问题制定SOP；③ 每周复盘。衡量标准：月底投诉率数据。"

---

### D - Do（执行）

**核心任务：** 按计划实施

**关键步骤：**
1. 严格按照计划执行
2. 记录执行过程中的数据
3. 记录遇到的意外情况

**要点：**
- 执行过程中不要随意偏离计划
- 遇到问题先记录，不要立刻改方案（留给C阶段分析）
- 尽可能收集数据，为后续检查提供依据

**示例：**
> "第1周：建立投诉分类台账，开始记录。第2-3周：对排名前3的投诉类型制定SOP并培训。第4周：持续执行并收集数据。"

---

### C - Check（检查/总结）

**核心任务：** 对照计划，评估结果

**关键步骤：**
1. 对比实际结果与预期目标
2. 分析差距的原因
3. 识别哪些做法有效、哪些无效

**要点：**
- **这是总结的关键环节**
- 用数据说话，不凭感觉
- 不仅看结果，还要看过程

**示例：**
> "月底数据：投诉率从5%降至3.2%，未达到2%的目标。分析原因：① 物流问题的SOP执行不到位（培训覆盖率仅60%）；② 新增了一类此前未预见的投诉（产品包装破损）。有效的做法：客服响应速度的提升明显减少了投诉升级。"

---

### A - Act（改进/处理）

**核心任务：** 标准化成功经验，修正失败做法

**关键步骤：**
1. 将有效的做法标准化、制度化
2. 将未解决的问题纳入下一轮PDCA
3. 总结经验教训，形成知识积累

**要点：**
- 成功经验如果不标准化，就会流失
- 未达到目标不是失败，不总结才是失败
- 每一轮PDCA的A，就是下一轮PDCA的P

**示例：**
> "标准化：将客服响应速度提升方案写入SOP。下一轮重点：① 提高物流SOP培训覆盖率至100%；② 针对包装破损问题启动专项改进。新一轮PDCA目标：投诉率降至2%以下。"

## 循环图示

```
        ┌─────────────┐
        │   P (Plan)   │ ← 制定目标和方案
        │    计划      │
        └──────┬──────┘
               │
               ▼
        ┌─────────────┐
        │   D (Do)     │ ← 按计划执行
        │    执行      │
        └──────┬──────┘
               │
               ▼
        ┌─────────────┐
        │   C (Check)  │ ← 检查结果，分析差距
        │    检查      │
        └──────┬──────┘
               │
               ▼
        ┌─────────────┐
        │   A (Act)    │ ← 标准化成功，改进不足
        │    改进      │
        └──────┬──────┘
               │
               ▼
         进入下一轮PDCA ──→ 持续循环 ──→ 螺旋上升
```

## PDCA的进阶理解

### 螺旋上升，不是原地打转

每完成一轮PDCA，质量就提升一个层次。这不是简单的循环，而是**螺旋式上升**：

```
第1轮PDCA → 投诉率 5% → 3.2%
第2轮PDCA → 投诉率 3.2% → 1.8%
第3轮PDCA → 投诉率 1.8% → 0.9%
...
```

### 大环套小环

一个大PDCA中可以嵌套多个小PDCA：

```
年度PDCA（大环）
  └─ Q1 PDCA（中环）
       └─ 每周PDCA（小环）
```

## 应用场景

| 场景 | 说明 |
|------|------|
| **质量管理** | PDCA最初的应用领域，产品质量持续改进 |
| **项目管理** | 每个迭代周期都是一个PDCA |
| **个人成长** | 月度/季度目标的制定、执行、复盘、调整 |
| **流程优化** | 反复循环，不断优化工作流程 |
| **学习计划** | 制定学习计划 → 学习 → 测试检验 → 调整方法 |

---

## PDCA与总结的关系

PDCA中的 **C（检查）** 和 **A（改进）** 环节本质上就是"总结"：

- **C（检查）** = 回顾发生了什么，与预期对比
- **A（改进）** = 提炼经验，制定下一步行动

> **PDCA让总结不再是偶发行为，而是嵌入日常工作节奏中的习惯。**

# 棱镜总结法（PRISM）

## 概述

棱镜总结法（PRISM）是一种融合了国内外18种经典总结方法论的综合性总结框架。其核心理念来自一个隐喻：

> **经历如白光，总结如棱镜——将混沌的经历折射为清晰、可复用的光谱。**

**PRISM** 五个字母，对应五个步骤：

| 字母 | 英文 | 中文 | 融合来源 |
|------|------|------|----------|
| **P** | Prepare | 准备（明确目标） | 总结五步流程、PDCA的Plan |
| **R** | Record | 记录（收集素材） | 渐进式总结Layer 1、AAR的"实际发生了什么" |
| **I** | Interpret | 解读（提炼加工） | Brown六步法、费曼技巧、5W1H、Gibbs反思循环 |
| **S** | Structure | 结构（系统组织） | 康奈尔笔记法、KPT、GRAI、STAR |
| **M** | Mobilize | 活化（指导行动） | Kolb循环、PDCA的Act、AAR |

---

## 核心理念

> ① **总结不是终点，是起点。**
> 好的总结不是"画句号"，而是"画箭头"——指向下一步行动。

> ② **总结不追求完美，追求有用。**
> 一份3分钟但有行动项的总结，远胜过一份3小时但无人看的"完美报告"。

> ③ **总结是复利投资，不是一次性消费。**
> 每次总结都归档到知识库，定期回看。一年后你会发现，自己早已站在了过去的肩膀上。

---

## Step 1 — Prepare（准备）：先瞄准，再开枪

> **没有目标的总结 = 没有棱镜的白光 = 依然是混沌。**

### 做什么？

在动笔之前，花 **3分钟** 回答三个问题：

| 问题 | 目的 | 示例 |
|------|------|------|
| **① 这次总结的目的是什么？** | 确定方向 | 汇报？复盘？学习？归档？ |
| **② 读者是谁？** | 确定深度和语言 | 给老板看？给自己看？给团队看？ |
| **③ 总结的时间边界是什么？** | 确定范围 | 一周？一个项目？一次阅读？ |

### 关键原则

- **一次总结只服务一个核心目的**。如果同时要汇报+复盘+学习，拆成三份。
- 目的决定了你**取舍的标尺**——与目标无关的内容，再精彩也要砍掉。

> 💡 灵感来源：总结五步流程"明确目标"、PDCA的Plan阶段

---

## Step 2 — Record（记录）：无数据，不总结

> **巧妇难为无米之炊。没有素材的总结只能是空谈。**

### 做什么？

根据目标，收集一切相关素材：

| 素材类型 | 来源 | 方法 |
|----------|------|------|
| **客观数据** | 日志、记录、数据报表 | 全量收集，不做筛选 |
| **主观感受** | 当时的情绪、直觉 | 及时记录，避免遗忘 |
| **关键事件** | 转折点、意外、亮点 | 标记"为什么重要" |
| **他人反馈** | 同事/朋友/导师的评价 | 原话记录，不加工 |

### 关键原则

- 📌 **先记录，不评判**——记录阶段不做筛选，避免遗漏
- 📌 **可借助工具**——日记、笔记App、录音、截图，不拘形式
- 📌 **设定期限**——不超过总时长的10%，避免陷入"收集癖"

> 💡 灵感来源：Tiago Forte 渐进式总结的 **Layer 1**（原始笔记）；AAR的"实际发生了什么"

---

## Step 3 — Interpret（解读）：从"发生了什么"到"意味着什么"

> **这是棱镜折射的核心——区分记录和总结的分水岭。**

这一步分 **三道加工工序**：

---

### 工序①：压缩提炼（Filter）

借鉴 **Ann Brown 六步法** 的精髓：

```
删除琐碎 → 删除冗余 → 用上位概念替代列表 → 提取/自创主题句
```

- 问自己：**如果只保留20%的内容，留什么？**
- 将大量细节压缩为**关键概念和核心命题**

---

### 工序②：多维解读（Lens）

借鉴 **5W1H + 多维度原则**，从至少 **3个不同维度** 解读同一件事：

| 维度 | 提问 |
|------|------|
| 🔍 事实维度 | 实际发生了什么？（What） |
| 🎯 目标维度 | 与预期有何差距？（Why） |
| 🛠️ 方法维度 | 做对了什么？做错了什么？（How） |
| 👥 人际维度 | 他人的反应说明了什么？（Who） |
| 📈 趋势维度 | 这件事在更大的图景中意味着什么？（So What） |

> **核心法则：一个维度只产出1-3条洞察，多维度交叉才能看到全貌。**

---

### 工序③：深度追问（Probe）

借鉴 **费曼技巧 + Gibbs反思循环**，用**三层追问**逼出底层认知：

```
第1层追问：为什么？（表因）
  → "因为预算不够"
第2层追问：为什么预算不够？（中因）
  → "因为没有提前做费用规划"
第3层追问：为什么没做规划？（根因）
  → "因为对这个项目的复杂度预估不足，下次需要……"
```

> 📌 **停下来的信号：当你的答案从"外部归因"转向"内部归因"——那就是真洞察。**

---

## Step 4 — Structure（结构）：让洞察变得可理解、可发现

> **零散的洞察不值钱，结构化的洞察才是知识。**

### 根据不同目的，选择不同的结构模板：

---

### 📊 模板A：工作/项目复盘 — **FACT框架**

| 模块 | 内容 | 字数占比 |
|------|------|----------|
| **F** - Facts（事实） | 关键数据、里程碑、结果 | 20% |
| **A** - Analysis（分析） | 差距在哪？根因是什么？ | 40% |
| **C** - Conclusion（结论） | 提炼出3条核心规律 | 20% |
| **T** - Todo（行动） | 下一步具体行动项 | 20% |

---

### 📖 模板B：学习/阅读总结 — **SQ3R变体**

```
S — Survey（概览）：一句话说这本书/课程讲了什么
Q — Questions（问题）：读之前/后的核心问题
    — Key Insights（关键洞察）：3-5个最大收获
    — Connections（连接）：与已有知识的关联
R — Retell（复述）：用自己的话讲给别人听
```

---

### 🌱 模板C：个人成长/周月总结 — **KPT+ 进化版**

| 模块 | 提问 |
|------|------|
| **🌟 Highlights（亮点）** | 这段时间最值得骄傲的3件事 |
| **📊 Keep（保持）** | 哪些做法有效，要继续？ |
| **⚠️ Problem（问题）** | 遇到了哪些问题/瓶颈？ |
| **🔬 Try（尝试）** | 下一步准备尝试什么新做法？ |
| **🧭 Direction（方向）** | 对长期目标有什么新的认识？ |

---

### 💾 模板D：知识归档 — **渐进式压缩**

借鉴 Tiago Forte 的理念，做**四级压缩**：

```
Level 1: 原文标注/原始素材（不动）
Level 2: 加粗关键段落（10秒）
Level 3: 高亮核心句（30秒）
Level 4: 一段话总结（2分钟）
Level 5: 一句话金句（随时可调用）
```

> 📌 **总结不是写一次就完事，而是随着理解加深不断迭代的。**

---

## Step 5 — Mobilize（活化）：总结不转化为行动 = 白总结

> **知识不用就是信息，信息不用就是数据，数据不看就是噪音。**

### 做什么？确保总结产出三类成果：

| 成果类型 | 说明 | 示例 |
|----------|------|------|
| **① 行动项** | 具体的、可执行的下一步 | "下次做项目前，先写一页费用规划" |
| **② 原则/规则** | 可复用的判断标准 | "住宿费不超过总预算的30%" |
| **③ 检查清单** | 标准化的流程清单 | "项目启动前的5项检查" |

### 关键原则

- 📌 每条行动项必须是 **动词开头**，有明确时间节点
- 📌 好的原则/规则要能回答：**"下次遇到类似情况，我直接怎么做？"**
- 📌 总结完成后，设置一个 **"回看提醒"**（1周后、1月后），让总结真正被使用

> 💡 灵感来源：Kolb循环的"主动实验"阶段；AAR的"我们能学到什么"

## 总览流程图

```
    ┌──────────┐
    │ 🎯 目标   │  P — Prepare
    │ (Prepare) │  明确目的、读者、边界
    └────┬─────┘
         │
         ▼
    ┌──────────┐
    │ 📝 记录   │  R — Record
    │ (Record)  │  收集数据、感受、事件、反馈
    └────┬─────┘
         │
         ▼
    ┌──────────┐
    │ 🔍 解读   │  I — Interpret
    │(Interpret)│  压缩 → 多维 → 追问
    └────┬─────┘
         │
         ▼
    ┌──────────┐
    │ 🧩 结构   │  S — Structure
    │(Structure)│  选模板、填框架、做压缩
    └────┬─────┘
         │
         ▼
    ┌──────────┐
    │ 🚀 活化   │  M — Mobilize
    │(Mobilize) │  行动项 + 原则 + 清单
    └──────────┘
```

## 不同场景速查表

| 场景 | 用哪些步骤 | 推荐模板 | 时间建议 |
|------|-----------|----------|----------|
| 📰 读完一篇文章 | P → I（压缩） → S | SQ3R变体 | 5-10分钟 |
| 📚 读完一本书 | P → R → I → S → M | 渐进式压缩 | 30-60分钟 |
| 🏗️ 项目结束复盘 | P → R → I → S → M（全套） | FACT框架 | 1-2小时 |
| 📅 每周个人总结 | P → R → I → S → M | KPT+ | 15-20分钟 |
| 🎓 学完一门课程 | R → I → S → M | SQ3R变体 + 渐进式 | 1-2小时 |
| ⚡ 一次快速反思 | P → I（追问） → M | 3-2-1策略 | 3-5分钟 |
| 💼 年终工作汇报 | P → R → I → S | FACT框架 | 半天-1天 |
